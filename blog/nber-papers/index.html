Next, letâ€™s set up our feature engineering. We will need to transform our text data into features useful for our model by tokenizing and computing (in this case) tf-idf. Letâ€™s also downsample since our dataset is imbalanced, with many more of some of the categories than others.

library(themis)
library(textrecipes)

nber_rec <-
  recipe(program_category ~ year + title, data = nber_train) %>%
  step_tokenize(title) %>%
  step_tokenfilter(title, max_tokens = 200) %>%
  step_tfidf(title) %>%
  step_downsample(program_category)

Since the lasso regularization penalty is a hyperparameter of the model (we canâ€™t find the best value from fitting the model a single time), letâ€™s tune over a grid of possible penalty parameters.

doParallel::registerDoParallel()

Choose and evaluate a final model

We could use the numerically best model with select_best() by often with regularized models we would rather choose a simpler model within some limits of performance. We can choose using the â€œone-standard error ruleâ€ with select_by_one_std_err() and then use last_fit() to fit one time to the training data and evaluate one time to the testing data.

  nber_rs %>%
  select_by_one_std_err(metric = "roc_auc", desc(penalty))
